# 游戏陪玩平台管理系统 - 启动指南

## 🚀 快速启动

### 1. 启动后端服务器
```bash
cd backend/server
node src/index.js
```
**后端服务地址**: http://localhost:3000

### 2. 启动前端服务器
```bash
cd frontend
npx vite --host --port 3004
```
**前端服务地址**: http://localhost:3004

## ⚠️ 重要提示

**前端启动问题解决方案**：
- 如果使用 `npm run dev` 启动失败，请直接使用 `npx vite --host --port 3004` 命令
- 这是由于 package.json 中的脚本配置问题导致的
- 直接使用 vite 命令可以绕过这个问题

## 🔔 通知系统修复

已修复通知在不同标签页之间重复显示的问题：
- 创建了全局通知状态管理器 (`NotificationContext.tsx`)
- 使用 localStorage 跨标签页共享已显示的通知ID
- 防止同一通知在多个标签页中重复弹出 toast 提示

## 📋 项目概述

这是一个**游戏陪玩平台管理系统**，包含前端React应用和后端Node.js API服务。

### 🎯 主要功能
- 👥 用户管理 (用户、陪玩、管理员)
- 📦 订单管理
- 🎮 游戏管理  
- 🎁 礼物系统
- 💰 提现管理
- 📊 统计分析
- 🔔 通知系统
- 🔐 权限管理

## 🔧 技术栈

### 前端 (Frontend)
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **UI框架**: Tailwind CSS
- **路由**: React Router DOM v7
- **状态管理**: React Context (AuthContext)
- **图表库**: Recharts
- **通知**: Sonner
- **表单**: React Hook Form + Zod
- **端口**: 3004

### 后端 (Backend)
- **运行时**: Node.js + Express
- **语言**: TypeScript (编译为JavaScript)
- **数据库**: MySQL 8.0
- **数据库驱动**: mysql2 (原生SQL)
- **认证**: JWT + bcrypt
- **文件上传**: multer
- **跨域**: CORS
- **环境配置**: dotenv
- **端口**: 3000

## 🛠️ 详细启动步骤

### 1. 环境准备
- Node.js 16+ 
- MySQL 8.0
- pnpm (推荐) 或 npm

### 2. 数据库设置
```sql
-- 1. 创建数据库
CREATE DATABASE author_center;

-- 2. 执行初始化脚本
mysql -u root -p author_center < backend/sql/init.sql

-- 3. 导入测试数据 (可选)
mysql -u root -p author_center < backend/sql/seed_data.sql
```

### 3. 后端启动
```bash
# 进入后端目录
cd backend/server

# 安装依赖 (如果还没安装)
npm install

# 编译TypeScript (如果需要)
npx tsc

# 启动服务器
node src/index.js
```

**启动成功标志**:

## 🐛 常见问题

### 1. 端口冲突
如果端口被占用，可以修改启动命令：
```bash
# 后端使用其他端口
PORT=3001 node src/index.js

# 前端使用其他端口  
npm run dev -- --port 3005
```

### 2. 数据库连接失败
检查 `.env` 文件中的数据库配置是否正确，确保MySQL服务已启动。

### 3. API请求失败
确认前后端都已正常启动，检查控制台是否有CORS错误。
